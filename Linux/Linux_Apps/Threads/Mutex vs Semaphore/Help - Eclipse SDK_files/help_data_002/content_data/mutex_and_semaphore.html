Mutex and Semaphore

Mutual exclusion (Osp::Base::Runtime::Mutex) instances allow multiple threads to access a shared resource one at a time in a thread safe manner. A Mutex locks an object for exclusive access by a thread. The threads that want to access a Mutex-locked object are queued until the Mutex is released.

Osp::Base::Runtime::Semaphores allow a limited number of threads to access a resource, and keep track of how many threads are accessing that resource. For example, if N printers are attached to one printer server, then the printer server can serve N printing tasks simultaneously.

Mutex and Semaphore resources are acquired and released using the Acquire() and Release() methods. The following code snippet demonstrates the implementation of these methods.

copytextpop-up
  • // Acquire and release a mutex
  • Mutex* pMutex = new Mutex;
  • pMutex->Create(); 
  • pMutex->Acquire();
  • // Use shared resource
  • pMutex->Release();
  •  
  • // Acquire and release a semaphore
  • Semaphore* pSemaphore = new Semaphore;
  • pSemaphore->Create(); 
  • pSemaphore->Acquire();
  • // Use shared resource
  • pSemaphore->Release();

Mutex* pMutex = new Mutex;
pMutex->Create(); 
pMutex->Acquire();
// Use shared resource
pMutex->Release();

// Acquire and release a semaphore
Semaphore* pSemaphore = new Semaphore;
pSemaphore->Create(); 
pSemaphore->Acquire();
// Use shared resource
pSemaphore->Release();
