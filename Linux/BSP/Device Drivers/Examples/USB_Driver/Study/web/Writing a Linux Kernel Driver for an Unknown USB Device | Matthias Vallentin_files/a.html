                                  Thanks for this post, I've been working on figuring out linux drivers and this looks to be a good example. Thank you!

                                  It looks like this init_MUTEX was removed somewhere in the 2.6.x kernel series, and I'm building 3.2.21. If anyone seems to have problems with compiling (I'm doing this for the relatively new raspberry pi) and gets a message about init_MUTEX, I did the following near the top of the file to fix it, just #define another alias for the same thing in the kernel, it seems they deprecated init_MUTEX. Put the following near the top somewhere after the #include statements.

                                  #ifndef init_MUTEX
                                  #define init_MUTEX(x) sema_init(x, 1)
                                  #endif

                                                                                                                                                  Thanks for this post, I've been working on figuring out linux drivers and this looks to be a good example. Thank you!

                                                                                                                                                  If anyone seems to have problems with compiling (I'm doing this for the relatively new raspberry pi) and gets a message about init_MUTEX, I did the following near the top of the file to fix it, just #define another alias for the same thing in the kernel, it seems they deprecated init_MUTEX

                                                                                                                                                  #ifndef init_MUTEX
                                                                                                                                                  #define init_MUTEX(x) sema_init(x, 1)
                                                                                                                                                  #endif
